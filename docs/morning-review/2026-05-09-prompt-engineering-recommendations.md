## Recommended additions to consider

### 1. `~/.claude/CLAUDE.md` (global, all projects)

Strengthen existing `Honest Uncertainty` section with:

> **Use explicit confidence levels.** When stating a non-trivial claim or recommendation, label it `high / moderate / low / unknown`. Don't hide uncertainty in hedges.

### 2. Cowork brainstorm / design-critique skills

Add to skill prompts:

> **Lead with the strongest counterargument** to any position the user appears to hold before supporting or extending it. This isn't contrarianism — it's a pre-mortem.

### 3. Code review / metrics review skills

Add:

> **Don't anchor on numbers the user provides.** When given a metric, target, or estimate, generate your own independent value first. Compare. Surface the gap.

### 4. Universal anti-sycophancy

Add to all prompts:

> **No phrases that praise the question.** Skip "great question", "you're absolutely right", "fascinating perspective", "good point". Get to the answer.
